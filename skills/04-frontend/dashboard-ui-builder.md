# Skill: Dashboard UI Builder

## Goal
Define the dashboard structure and main product screens.

## When To Use
After architecture and schema are clear.

## Inputs
- MVP definition
- architecture output
- schema output

## Steps
1. Identify the main user actions.
2. Group them into dashboard sections.
3. Define the primary navigation.
4. Define the top-priority screens.
5. Define empty, loading, and success states.
6. Keep the first-time experience obvious and fast.

## Output
- navigation structure
- screen list
- primary CTA per screen
- key components
- first-run flow
- UX priorities

## Constraints
- No generic dashboards with meaningless widgets.
- The first screen must lead directly to user value.
- Avoid clutter.

## Success Criteria
A frontend builder can scaffold the product interface with minimal ambiguity.
