# Skill: MVP Definer

## Goal
Define the smallest sellable version of the product.

## When To Use
After classification and market analysis.

## Inputs
- `project-input/idea.md`
- `project-input/constraints.md`
- market analysis output

## Steps
1. Identify the one core promise of the product.
2. Select the minimum set of features required to deliver that promise.
3. Remove any feature that does not support first value delivery.
4. Define what will be excluded from v1.
5. Define one success metric for the MVP.

## Output
- core promise
- target user
- MVP features
- excluded features
- primary workflow
- success metric

## Constraints
- Prefer 3 to 5 core features.
- Exclude enterprise-grade complexity from v1.
- Avoid multi-role complexity unless essential.

## Success Criteria
The MVP is narrow, useful, and realistically buildable in a short cycle.
