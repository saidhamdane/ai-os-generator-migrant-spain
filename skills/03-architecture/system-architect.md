# Skill: System Architect

## Goal
Design a clean, implementation-ready architecture for the MVP.

## When To Use
After MVP definition.

## Inputs
- MVP definition
- `project-input/constraints.md`

## Steps
1. Identify core modules.
2. Define frontend, backend, database, and external services.
3. Map user actions to system components.
4. Define integration boundaries.
5. Recommend the simplest production-worthy architecture.

## Output
- architecture summary
- frontend responsibilities
- backend responsibilities
- database role
- third-party integrations
- key data flows
- deployment shape

## Constraints
- Keep architecture small and practical.
- Avoid microservices for MVP unless absolutely necessary.
- Align with default stack unless justified.

## Success Criteria
A developer can use this architecture as the base for implementation without major redesign.
