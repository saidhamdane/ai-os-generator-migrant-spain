# Execution Rules

## Goal
Keep project generation consistent, practical, and shippable.

## Rules
1. Start from business value, not code.
2. One phase at a time.
3. Every recommendation must support:
   - speed
   - monetization
   - simplicity
   - maintainability
4. Prefer version-one practicality over perfection.
5. Explicitly list what is not included in the MVP.
6. Keep deliverables structured and reusable.
7. Name files, entities, and modules clearly.
8. Use default stack unless a strong reason exists to change it.
9. Avoid overengineering.
10. Outputs should be ready for implementation or handoff.

## Red Flags
- too many features in MVP
- unclear target user
- architecture without business rationale
- premium pricing without clear value
- generic dashboard with no workflow logic
